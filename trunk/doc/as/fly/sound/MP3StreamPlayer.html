Property detail
framesPerSWFproperty
public var framesPerSWF:uint

The amount of mp3 frames that will be written to each swf. Make sure this value is larger then 2 times the overlap.

If you lower this value, more swf's need to be generated, but the time it takes to extract the sound from each swf will go down.

This value can be seen as the primary buffer.

If the sound is not buffering good enough you can enable the soundBuffer

The default value is 250.

See also

soundBuffer
overlapproperty 
overlap:uint  [read-write]

The actual MP3 data is not bound to a single MP3 frame, but overlaps multiple frames. For this reason we need to write more bytes to an swf than we actually use. This variable determines the overlap.

Raising the overlap helps removing strange clicks.

You can not set the overlap higher then 7.

The default value is 7.

soundBufferSizeproperty 
public var soundBufferSize:uint

If you set this propery, the actual playing of the sound will start after the actual sound buffer contains the given amount of entries.

Set this property if you get alot of BufferEvent.BUFFER_EMPTY events.

Raising this property will increase the delay between buffering and playing.
